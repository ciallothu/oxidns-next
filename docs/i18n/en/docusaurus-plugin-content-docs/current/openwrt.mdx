OxiDNS Next can run portably on OpenWrt devices whose CPU architecture has a matching release. The current distribution does not include a dedicated LuCI app and does not automatically register an init service on OpenWrt.

<Admonition type="warning" title="Upstream LuCI boundary">

[`luci-app-oxidns`](https://github.com/svenshi/luci-app-oxidns) accompanies upstream OxiDNS. Its default repository, binary name, and filesystem paths target the original distribution. It is not an OxiDNS Next installer and should not be used directly to manage this project.

</Admonition>

## Supported devices

The installer currently recognizes these Linux architectures:

| `uname -m` | Release target |
| --- | --- |
| `x86_64` / `amd64` | `x86_64-unknown-linux-musl` |
| `aarch64` / `arm64` | `aarch64-unknown-linux-musl` |
| `i386` / `i686` | `i686-unknown-linux-musl` |
| `armv7` / `armv7l` | `arm-unknown-linux-musleabihf` |

Unlisted architectures such as MIPS and MIPS64 do not currently have official OxiDNS Next release archives. Do not force an incompatible build through `OXIDNS_NEXT_TARGET`.

## Install

Run as root:

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sh
```

If `curl` is unavailable:

```sh
wget -O- https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sh
```

The default paths are:

- Binary: `/opt/oxidns-next/oxidns-next`
- Command link: `/usr/local/bin/oxidns-next`
- Configuration: `/opt/oxidns-next/config.yaml`
- WebUI: `/opt/oxidns-next/webui`

On OpenWrt, the installer behaves as if service installation were disabled. Start OxiDNS Next yourself through procd, another process manager, or a foreground command:

```sh
oxidns-next start -c /opt/oxidns-next/config.yaml -d /opt/oxidns-next
```

Environment variables can select a version or custom directories:

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh \
  | env OXIDNS_NEXT_VERSION=v0.1.0 \
        OXIDNS_NEXT_INSTALL_DIR=/opt/oxidns-next \
        OXIDNS_NEXT_BIN_DIR=/usr/bin \
        sh
```

## Upgrade and uninstall

Run the installer again to replace the binary and WebUI. An existing `config.yaml` is retained and the new example is written to `config.yaml.example`.

Uninstall while keeping the configuration:

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh | sh
```

Back up the configuration before removing the complete install directory:

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh \
  | env OXIDNS_NEXT_PURGE=1 sh
```

## Resource guidance

The router must be able to reach GitHub Releases. For resource-constrained devices, validate memory use, configuration, and the selected plugin set on a matching test device before moving it onto the gateway. `minimal` and `standard` slim archives are currently published only for x86_64 and aarch64 musl targets.
