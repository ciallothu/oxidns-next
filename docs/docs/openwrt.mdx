OxiDNS Next 可以在受支持 CPU 架构的 OpenWrt 设备上以便携方式运行。当前发行版不包含专用 LuCI 插件，也不会在 OpenWrt 上自动注册 init 服务。

<Admonition type="warning" title="与上游 LuCI 插件的边界">

[`luci-app-oxidns`](https://github.com/svenshi/luci-app-oxidns) 是上游 OxiDNS 的配套项目，其默认仓库、二进制名称和路径均指向原始发行版。它不是 OxiDNS Next 的安装器，不应直接用来管理本项目。

</Admonition>

## 支持的设备

安装脚本目前可自动识别以下 Linux 架构：

| `uname -m` | Release 目标 |
| --- | --- |
| `x86_64` / `amd64` | `x86_64-unknown-linux-musl` |
| `aarch64` / `arm64` | `aarch64-unknown-linux-musl` |
| `i386` / `i686` | `i686-unknown-linux-musl` |
| `armv7` / `armv7l` | `arm-unknown-linux-musleabihf` |

MIPS、MIPS64 等未列出的架构当前没有官方 OxiDNS Next release archive。请勿通过 `OXIDNS_NEXT_TARGET` 强行选择不兼容的目标。

## 安装

以 root 用户执行：

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sh
```

没有 `curl` 时：

```sh
wget -O- https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sh
```

脚本默认把文件安装到：

- 二进制：`/opt/oxidns-next/oxidns-next`
- 命令链接：`/usr/local/bin/oxidns-next`
- 配置：`/opt/oxidns-next/config.yaml`
- WebUI：`/opt/oxidns-next/webui`

OpenWrt 会自动使用 `OXIDNS_NEXT_INSTALL_SERVICE=0` 的等效行为。安装结束后需要自行使用 procd、其他进程管理器或前台命令启动：

```sh
oxidns-next start -c /opt/oxidns-next/config.yaml -d /opt/oxidns-next
```

可通过环境变量调整目录或版本：

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh \
  | env OXIDNS_NEXT_VERSION=v0.1.0 \
        OXIDNS_NEXT_INSTALL_DIR=/opt/oxidns-next \
        OXIDNS_NEXT_BIN_DIR=/usr/bin \
        sh
```

## 升级与卸载

再次运行安装脚本即可替换二进制和 WebUI。已有 `config.yaml` 会保留，新版本示例写入 `config.yaml.example`。

卸载时默认保留配置：

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh | sh
```

彻底删除安装目录前请先备份配置：

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh \
  | env OXIDNS_NEXT_PURGE=1 sh
```

## 资源建议

路由器需要能够访问 GitHub Releases。资源紧张的设备建议先在同架构测试环境验证内存占用、配置和插件组合，再迁移到网关；`minimal` 与 `standard` 精简 archive 目前仅为 x86_64 和 aarch64 musl 目标发布。
