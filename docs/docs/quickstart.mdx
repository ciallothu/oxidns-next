本页介绍 OxiDNS Next 的安装方式，并给出从下载到首次启动的最短路径。

<Admonition type="tip" title="选哪种安装方式？">

* **Linux 服务器**：优先使用一条命令安装脚本、release 压缩包或 `.deb`
* **OpenWrt 路由器**：使用与设备架构匹配的 musl archive；当前为[便携部署](openwrt.mdx)，不含专用 LuCI
* **容器环境**：优先使用 Docker 镜像
* **需要自行改代码或调试**：使用 Cargo 从源码构建
* **macOS / Windows**：优先使用一条命令安装脚本或 release 压缩包

</Admonition>

## 1. 一条命令安装

安装脚本会自动识别当前平台，下载匹配的 OxiDNS Next GitHub Release archive，并解压 `oxidns-next` / `oxidns-next.exe`、`config.yaml` 和 `webui/`。Linux、macOS 与管理员模式的 Windows 默认安装并启动系统服务；OpenWrt 只执行便携安装。

<Tabs groupId="os">
  <TabItem value="unix" label="Linux / macOS" default>

```bash
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sudo sh
```

  </TabItem>
  <TabItem value="windows" label="Windows">

```powershell
irm https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.ps1 | iex
```

  </TabItem>
  <TabItem value="openwrt" label="OpenWrt">

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sh
```

没有 `curl` 时：

```sh
wget -O- https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sh
```

  </TabItem>
</Tabs>

默认安装位置：

| 平台 | 默认安装目录 | PATH 行为 |
| --- | --- | --- |
| Linux / macOS `sudo` 服务安装 | `/opt/oxidns-next` | 在 `/usr/local/bin/oxidns-next` 创建链接，并安装、启动系统服务 |
| Windows 管理员 PowerShell 服务安装 | `%ProgramFiles%\OxiDNS Next` | 加入 Machine PATH，并安装、启动系统服务 |
| Linux / macOS 便携安装 | `~/.oxidns-next` | 设置 `OXIDNS_NEXT_INSTALL_SERVICE=0` 后，在 `~/.local/bin/oxidns-next` 创建链接 |
| Windows 便携安装 | `%LOCALAPPDATA%\OxiDNS Next` | 设置 `OXIDNS_NEXT_INSTALL_SERVICE=0` 后，将安装目录加入用户 PATH |
| OpenWrt 便携安装 | `/opt/oxidns-next` | 在 `/usr/local/bin/oxidns-next` 创建链接，不自动注册 init 服务 |

在 Linux / macOS / Windows archive 安装模式下，重复运行安装脚本会更新二进制和 `webui/`。如果安装目录中已经有 `config.yaml`，脚本会保留现有配置，并把 release 内置配置写成 `config.yaml.example`。

在 OpenWrt 上重复运行安装脚本会更新二进制和 WebUI，并保留现有 `config.yaml`；服务托管需要自行配置。

### 安装指定版本

<Tabs groupId="os">
  <TabItem value="unix" label="Linux / macOS" default>

```bash
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sudo env OXIDNS_NEXT_VERSION=v0.1.0 sh
```

  </TabItem>
  <TabItem value="windows" label="Windows">

```powershell
$env:OXIDNS_NEXT_VERSION = "v0.1.0"; irm https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.ps1 | iex
```

  </TabItem>
</Tabs>

常用环境变量：

| 变量 | 作用 |
| --- | --- |
| `OXIDNS_NEXT_VERSION` | release 标签，默认 `latest` |
| `OXIDNS_NEXT_INSTALL_DIR` | 覆盖安装目录 |
| `OXIDNS_NEXT_BIN_DIR` | Linux / macOS 下覆盖命令链接目录 |
| `OXIDNS_NEXT_TARGET` | 手动指定 release target，例如 `x86_64-unknown-linux-musl` |
| `OXIDNS_NEXT_BUNDLE` | 选择 release 编译版本，默认 `full`；Linux x86_64 / ARM64 musl 可选 `minimal` 或 `standard` |
| `OXIDNS_NEXT_NO_PATH=1` | 不创建命令链接或不修改 PATH |
| `OXIDNS_NEXT_INSTALL_SERVICE=0` | 不安装系统服务，改为便携安装 |
| `OXIDNS_NEXT_START_SERVICE=0` | 安装服务但不立即启动 |

便携安装：

<Tabs groupId="os">
  <TabItem value="unix" label="Linux / macOS" default>

```bash
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | env OXIDNS_NEXT_INSTALL_SERVICE=0 sh
```

  </TabItem>
  <TabItem value="windows" label="Windows">

```powershell
$env:OXIDNS_NEXT_INSTALL_SERVICE = "0"; irm https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.ps1 | iex
```

  </TabItem>
</Tabs>

便携安装完成后可直接运行：

<Tabs groupId="os">
  <TabItem value="unix" label="Linux / macOS" default>

```bash
oxidns-next start -c ~/.oxidns-next/config.yaml -d ~/.oxidns-next
```

  </TabItem>
  <TabItem value="windows" label="Windows">

```powershell
oxidns-next.exe start -c "$env:LOCALAPPDATA\OxiDNS Next\config.yaml" -d "$env:LOCALAPPDATA\OxiDNS Next"
```

  </TabItem>
</Tabs>

<Admonition type="note" title="端口权限">

默认配置包含 `:53` 监听。在 Linux / macOS 上，便携安装的普通用户通常不能绑定 53 端口；仅做本地验证时可先移除 `udp_server_53`，或使用默认的系统服务安装。

</Admonition>

### 卸载

卸载脚本会移除二进制、`webui/` 和 PATH 入口，默认保留 `config.yaml`。OpenWrt 使用相同的便携卸载逻辑。

<Tabs groupId="os">
  <TabItem value="unix" label="Linux / macOS" default>

```bash
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh | sudo sh
```

  </TabItem>
  <TabItem value="openwrt" label="OpenWrt">

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh | sh
```

没有 `curl` 时：

```sh
wget -O- https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh | sh
```

  </TabItem>
  <TabItem value="windows" label="Windows">

```powershell
irm https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.ps1 | iex
```

  </TabItem>
</Tabs>

如果安装时使用了 `sudo` 或自定义 `OXIDNS_NEXT_INSTALL_DIR`，卸载时也请保持相同权限和目录变量。OpenWrt 卸载需要 root。

彻底清理安装目录和配置：

<Tabs groupId="os">
  <TabItem value="unix" label="Linux / macOS" default>

```bash
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh | sudo env OXIDNS_NEXT_PURGE=1 sh
```

  </TabItem>
  <TabItem value="openwrt" label="OpenWrt">

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh | env OXIDNS_NEXT_PURGE=1 sh
```

没有 `curl` 时：

```sh
wget -O- https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.sh | env OXIDNS_NEXT_PURGE=1 sh
```

  </TabItem>
  <TabItem value="windows" label="Windows">

```powershell
$env:OXIDNS_NEXT_PURGE = "1"; irm https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/uninstall.ps1 | iex
```

  </TabItem>
</Tabs>

<Admonition type="warning" title="彻底清理">

设置 `OXIDNS_NEXT_PURGE=1` 会删除整个安装目录，包括配置文件。请确认 `config.yaml` 已备份后再执行。

</Admonition>

常用卸载变量：

| 变量 | 作用 |
| --- | --- |
| `OXIDNS_NEXT_INSTALL_DIR` | 覆盖卸载目标目录，应与安装时一致 |
| `OXIDNS_NEXT_BIN_DIR` | Linux / macOS 下覆盖命令链接目录 |
| `OXIDNS_NEXT_NO_PATH=1` | 不移除命令链接或用户 PATH |
| `OXIDNS_NEXT_UNINSTALL_SERVICE=1` | 强制卸载系统服务；默认在 root / 管理员环境中自动尝试 |
| `OXIDNS_NEXT_PURGE=1` | 删除整个安装目录，包括配置文件 |

## 2. 从源码构建

适合需要调试、二次开发或暂时不依赖发行包的场景。

前提：

- 已安装稳定版 Rust 工具链
- 可用的 Cargo 构建环境

构建并运行：

```bash
git clone https://github.com/ciallothu/oxidns-next.git
cd oxidns-next

cargo build --release
./target/release/oxidns-next check -c config.yaml
./target/release/oxidns-next start -c config.yaml
```

调试运行：

```bash
cargo run -- check -c config.yaml
cargo run -- start -c config.yaml -l debug
```

## 3. 使用 GitHub Releases 二进制包

OxiDNS Next 的 release 工作流会为多个平台生成独立二进制压缩包。`full` 压缩包内默认包含：

- `oxidns-next` 或 `oxidns-next.exe`
- `config.yaml`
- `LICENSE`
- `webui/`

Linux x86_64 / ARM64 musl 额外提供 `minimal` 和 `standard` slim 压缩包。`minimal` 只包含 `oxidns-next`、`config.yaml` 和 `LICENSE`；`standard` 还包含 WebUI 静态文件，并内置 query_recorder 与 upgrade 能力。各预设编译版的完整能力差异见[自定义编译](./custom-build#预设能力矩阵)。

发布页地址：

- [https://github.com/ciallothu/oxidns-next/releases](https://github.com/ciallothu/oxidns-next/releases)

### 支持的压缩包目标

非 Windows 平台使用 `.tar.gz`：

- `oxidns-next-x86_64-unknown-linux-gnu.tar.gz`
- `oxidns-next-x86_64-unknown-linux-musl.tar.gz`
- `oxidns-next-aarch64-unknown-linux-gnu.tar.gz`
- `oxidns-next-aarch64-unknown-linux-musl.tar.gz`
- `oxidns-next-i686-unknown-linux-musl.tar.gz`
- `oxidns-next-arm-unknown-linux-musleabihf.tar.gz`
- `oxidns-next-x86_64-apple-darwin.tar.gz`
- `oxidns-next-aarch64-apple-darwin.tar.gz`
- `oxidns-next-x86_64-unknown-freebsd.tar.gz`

Linux musl slim 包使用 bundle 前缀：

- `oxidns-next-minimal-x86_64-unknown-linux-musl.tar.gz`
- `oxidns-next-minimal-aarch64-unknown-linux-musl.tar.gz`
- `oxidns-next-standard-x86_64-unknown-linux-musl.tar.gz`
- `oxidns-next-standard-aarch64-unknown-linux-musl.tar.gz`

Windows 平台使用 `.zip`：

- `oxidns-next-x86_64-pc-windows-msvc.zip`
- `oxidns-next-i686-pc-windows-msvc.zip`
- `oxidns-next-aarch64-pc-windows-msvc.zip`

### 不同系统如何选择 release 文件

无法确定应该下载哪个 asset 时，可按下面选择：

| 系统 / 环境 | 推荐 release 文件 | 说明 |
| --- | --- | --- |
| Linux x86_64 | `oxidns-next-x86_64-unknown-linux-musl.tar.gz` | 默认优先选这个，兼容性更稳 |
| Linux ARM64 | `oxidns-next-aarch64-unknown-linux-musl.tar.gz` | 默认优先选这个，兼容性更稳 |
| Debian / Ubuntu x86_64 服务安装 | `*_amd64.deb` | 适合 systemd 服务部署 |
| Debian / Ubuntu ARM64 服务安装 | `*_arm64.deb` | 适合 systemd 服务部署 |
| OpenWrt | 与设备 CPU 架构匹配的 Linux musl archive | 安装脚本以便携模式部署；MIPS / MIPS64 当前无官方 archive |
| Alpine Linux x86_64 | `oxidns-next-x86_64-unknown-linux-musl.tar.gz` | Alpine 建议优先选 musl |
| Alpine Linux ARM64 | `oxidns-next-aarch64-unknown-linux-musl.tar.gz` | 静态链接更省心 |
| 明确是 glibc Linux 且需要动态链接版本 | `oxidns-next-x86_64-unknown-linux-gnu.tar.gz` / `oxidns-next-aarch64-unknown-linux-gnu.tar.gz` | 仅在明确环境匹配时选择 |
| 32 位 ARM Linux | `oxidns-next-arm-unknown-linux-musleabihf.tar.gz` | 适合部分树莓派和老 ARM 板子 |
| macOS Intel | `oxidns-next-x86_64-apple-darwin.tar.gz` | Intel Mac |
| macOS Apple Silicon | `oxidns-next-aarch64-apple-darwin.tar.gz` | M1 / M2 / M3 / M4 |
| Windows x64 | `oxidns-next-x86_64-pc-windows-msvc.zip` | 常见 PC |
| Windows 32-bit | `oxidns-next-i686-pc-windows-msvc.zip` | 仅在 32 位 Windows 上使用 |
| Windows ARM64 | `oxidns-next-aarch64-pc-windows-msvc.zip` | ARM Windows 设备 |
| FreeBSD x86_64 | `oxidns-next-x86_64-unknown-freebsd.tar.gz` | FreeBSD 主机 |

如果还不确定，可先查看本机信息：

```bash
uname -s
uname -m
```

常见输出和目标的对应关系：

- `Linux` + `x86_64`：默认选 `x86_64-unknown-linux-musl`，只有明确需要 glibc 动态链接版本时再选 `x86_64-unknown-linux-gnu`
- `Linux` + `aarch64`：默认选 `aarch64-unknown-linux-musl`，只有明确需要 glibc 动态链接版本时再选 `aarch64-unknown-linux-gnu`
- `Linux` + `armv7l`：选 `arm-unknown-linux-musleabihf`
- `Darwin` + `x86_64`：选 `x86_64-apple-darwin`
- `Darwin` + `arm64`：选 `aarch64-apple-darwin`

Windows 可在 PowerShell 中执行：

```powershell
(Get-CimInstance Win32_OperatingSystem).OSArchitecture
```

输出为 `X64`、`Arm64` 或 `X86` 时，分别对应下载 `x86_64`、`aarch64`、`i686` 的 Windows release。

### Linux / macOS 安装示例

将下面的 `TAG` 替换为实际 release 标签，例如 `v0.1.0`。Linux 默认以 `x86_64-unknown-linux-musl` 为例：

```bash
curl -L -o oxidns-next.tar.gz \
  https://github.com/ciallothu/oxidns-next/releases/download/TAG/oxidns-next-x86_64-unknown-linux-musl.tar.gz

mkdir -p oxidns-next
tar -xzf oxidns-next.tar.gz -C oxidns-next
cd oxidns-next

chmod +x oxidns-next
./oxidns-next check -c config.yaml
./oxidns-next start -c config.yaml
```

无法确认目标机的 glibc 版本，或运行在 Alpine Linux 上时，应优先选择 `*-linux-musl` 版本，不要默认使用 `gnu` 版本。

### Windows 安装示例

下载对应 `zip` 后解压，在 PowerShell 中运行：

```powershell
.\oxidns-next.exe start -c .\config.yaml
```

## 4. 使用 Debian 包安装

release 工作流当前会额外生成以下 Debian 包：

- `x86_64-unknown-linux-gnu`
- `aarch64-unknown-linux-gnu`

安装：

```bash
sudo dpkg -i oxidns-next_*_amd64.deb
```

或在 ARM64 Debian / Ubuntu 上：

```bash
sudo dpkg -i oxidns-next_*_arm64.deb
```

安装后默认文件位置：

- 二进制：`/usr/bin/oxidns-next`
- 配置：`/etc/oxidns-next/config.yaml`
- 运行期相对路径基准：`/var/lib/oxidns-next`
- WebUI：`/var/lib/oxidns-next/webui` 软链接到 `/usr/share/oxidns-next/webui`

项目也包含 systemd 打包配置，因此在 Debian 系发行版上适合直接作为系统服务部署。默认服务通过 `ExecStart ... -d /var/lib/oxidns-next` 明确指定运行期相对路径基准。

默认配置被修改后，建议先校验一次：

```bash
oxidns-next check -c /etc/oxidns-next/config.yaml -d /var/lib/oxidns-next
```

验证服务状态：

```bash
sudo systemctl status oxidns-next
```

如果服务尚未启动，可执行：

```bash
sudo systemctl enable --now oxidns-next
```

## 5. OpenWrt 便携部署

OpenWrt 使用与 Linux 相同的 release archive，但安装脚本会跳过系统服务注册：

```sh
curl -fsSL https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sh
```

没有 `curl` 时：

```sh
wget -O- https://raw.githubusercontent.com/ciallothu/oxidns-next/main/scripts/install.sh | sh
```

安装完成后请自行通过 procd 或其他进程管理器托管 `oxidns-next`。上游 [`luci-app-oxidns`](https://github.com/svenshi/luci-app-oxidns) 面向原始 OxiDNS，不是本项目安装器。支持架构、默认路径和卸载说明见 [OpenWrt 部署](openwrt.mdx)。

## 6. 使用 Docker 镜像

仓库提供 Docker 镜像发布流程，镜像仓库地址为：

- `ghcr.io/ciallothu/oxidns-next`

当前 Docker 发布流程构建以下平台：

- `linux/amd64`
- `linux/arm64`

拉取并运行：

```bash
docker pull ghcr.io/ciallothu/oxidns-next:latest

# 容器中的浏览器请求不是 loopback，首次初始化必须使用临时引导 token。
export OXIDNS_NEXT_BOOTSTRAP_TOKEN="$(openssl rand -hex 32)"

docker run -d \
  --name oxidns-next \
  --restart unless-stopped \
  -p 53:5335/udp \
  -p 53:5335/tcp \
  -p 9199:9199/tcp \
  -e OXIDNS_NEXT_BOOTSTRAP_TOKEN \
  -v "$(pwd)/config.yaml:/etc/oxidns-next/config.yaml:ro" \
  -v "$(pwd)/data:/etc/oxidns-next/data" \
  ghcr.io/ciallothu/oxidns-next:latest
```

<Admonition type="note" title="端口绑定">

Linux 上 53 是特权端口；普通用户运行 Docker 时一般通过 Docker 守护进程拿到权限。便携安装非 root 用户应使用 `:5353`。

</Admonition>

如果默认分支镜像已发布，也可以使用 `latest` 标签。

镜像默认启动命令等价于：

```bash
oxidns-next start -c /etc/oxidns-next/config.yaml -d /etc/oxidns-next
```

镜像元数据声明暴露：

- `53/udp`
- `53/tcp`
- `9199/tcp`

仓库根目录的 `config.yaml` 为了便于非 root 本地运行，DNS 监听使用 `:5335`。上面的 `docker run` 和下面的 Compose 示例把宿主机 `53` 映射到容器 `5335`；如果你改成容器内监听 `:53`，端口映射也应改回 `53:53`。

### Docker Compose 示例

使用 Compose 管理配置和端口映射时，可参考下面的 `docker-compose.yml`：

```yaml
services:
  oxidns-next:
    image: ghcr.io/ciallothu/oxidns-next:latest
    container_name: oxidns-next
    restart: unless-stopped
    ports:
      - "53:5335/udp"
      - "53:5335/tcp"
      - "9199:9199/tcp"
    environment:
      - OXIDNS_NEXT_BOOTSTRAP_TOKEN
    volumes:
      - ./config.yaml:/etc/oxidns-next/config.yaml:ro
      - ./data:/etc/oxidns-next/data
```

启动：

```bash
export OXIDNS_NEXT_BOOTSTRAP_TOKEN="$(openssl rand -hex 32)"
docker compose up -d
```

首次打开 WebUI 时，在初始化表单中填入这个 token 并创建管理员。完成后立即移除容器中的临时 token：

```bash
unset OXIDNS_NEXT_BOOTSTRAP_TOKEN
docker compose up -d --force-recreate
```

直接使用 `docker run` 时也应在管理员创建后删除并按原参数重新创建容器，但不再传入 `-e OXIDNS_NEXT_BOOTSTRAP_TOKEN`。不要把 token 写入 `config.yaml`、Compose 文件或 shell 历史；若不使用 OpenSSL，请通过密码管理器生成至少 32 字节的随机值。

查看日志：

```bash
docker compose logs -f oxidns-next
```

默认单容器部署使用 SQLite 查询历史。需要 PostgreSQL、MySQL，或在 SQLite 之外启用 Redis DNS/API 缓存时，请使用仓库 [`examples/storage`](https://github.com/ciallothu/oxidns-next/tree/main/examples/storage) 中带健康检查、环境变量占位符和持久卷的 Compose 示例。Redis 不是查询日志的持久化数据源。

## 7. 选择建议

- 想最快验证功能：使用一条命令安装最新 release，或手动下载对应 release 压缩包
- 想以系统服务长期运行：优先使用 Debian 包
- 想在 OpenWrt 部署：先核对设备架构，再使用便携安装脚本并自行配置进程托管
- 想在容器平台部署：使用 GHCR 镜像
- 想参与开发或自行裁剪：从源码构建
